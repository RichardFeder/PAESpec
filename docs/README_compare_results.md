# Comparing PAE and Template Fitting Redshift Results

This guide explains how to use `compare_redshift_results.py` to streamline the analysis of redshift estimation results from `redshift_job.py` and produce comparison plots.

## Overview

The `compare_redshift_results.py` script automates the workflow of:
1. Loading PAE redshift results from batch processing (`redshift_job.py` outputs)
2. Loading template fitting (TF) results
3. Generating comparison plots from `paper_plots.ipynb`:
   - `compare_pae_tf_redshifts2`: Side-by-side PAE vs TF redshift scatter plots
   - `plot_coverage_comparison_grid`: Coverage comparison across uncertainty bins

## Quick Start

### Basic PAE vs TF Comparison Plot

```bash
python compare_redshift_results.py \
    --pae_results <path_to_PAE_results.npz> \
    --pae_samples <path_to_PAE_samples.npz>
```

### Include Coverage Analysis

```bash
python compare_redshift_results.py \
    --pae_results <path_to_PAE_results.npz> \
    --pae_samples <path_to_PAE_samples.npz> \
    --tf_load_zpdf \
    --plot_coverage
```

## Arguments

### Required
- `--pae_results`: Path to PAE redshift results file (`.npz` from `redshift_job.py`)
- `--pae_samples`: Path to PAE samples file (`.npz` from `redshift_job.py`)

### Optional
- `--tf_load_zpdf`: Load template fitting PDFs (required for coverage plots)
- `--output_dir`: Directory for output figures (default: `figures/comparison/`)
- `--nsrc`: Number of sources to analyze (default: all)
- `--src_idxs`: Specific source indices to analyze (space-separated)
- `--plot_comparison`: Generate PAE vs TF comparison plot (default: True)
- `--plot_coverage`: Generate coverage comparison grid (requires `--tf_load_zpdf`)
- `--sigz_bins`: Fractional uncertainty bin edges (default: 0.0 0.02 0.05 0.1 0.2)
- `--burn_in`: Number of burn-in steps to exclude from chains (default: 1000)

```bash

    --pae_samples $SCRATCH/data/pae_sample_results/MCLMC/samples/PAE_samples_500_srcs_*.npz \
```

```bash

    --pae_results path/to/results.npz \
    --pae_samples path/to/samples.npz \
    --src_idxs 10 25 50 75 100 \
    --output_dir figures/selected_sources/
```

## Output Files

The script generates PNG figures in the specified output directory:

1. **`pae_tf_comparison.png`**: Side-by-side scatter plots comparing PAE and TF redshift estimates vs true redshifts, color-coded by fractional uncertainty

2. **`coverage_comparison_grid.png`** (if `--plot_coverage`): Multi-row comparison grid showing:
   - Fractional error distributions
   - Z-score distributions  
   - Scatter plots (estimated vs true z)
   - PIT histograms
   
   Each column represents a different fractional uncertainty bin.

## Data Requirements

### PAE Results Files
Generated by `redshift_job.py`, containing:
- `ztrue`: True redshifts
- `z_mean`, `z_med`: Mean/median redshift estimates
- `err_low`, `err_high`: Uncertainty bounds
- `R_hat`: Gelman-Rubin convergence diagnostic
- `chi2`: Reduced chi-squared values

### PAE Samples Files
Generated by `redshift_job.py`, containing:
- `all_samples`: MCLMC posterior samples (n_src × n_chain × n_step × n_param)
- `ztrue`: True redshifts

### Template Fitting Results
Automatically loaded from:
- `data/tf_results/photoz_result_*.out`
- `data/tf_results/photoz_pdfs_*.out` (if `--tf_load_zpdf`)

## Integration with Existing Workflow

This script works seamlessly with your existing pipeline:

```bash
# 1. Run batch redshift estimation
python scripts/redshift_job.py --start_idx 0 --ngal 500

# 2. Compare results and generate plots  
python scripts/compare_redshift_results.py \
    --pae_results $SCRATCH/data/pae_sample_results/MCLMC/zres/PAE_results_500_srcs_*_start0.npz \
    --pae_samples $SCRATCH/data/pae_sample_results/MCLMC/samples/PAE_samples_500_srcs_*_start0.npz \
    --tf_load_zpdf \
    --plot_coverage \
    --output_dir figures/batch0_comparison/
```

## Notes

- The script uses the existing `prepare_data_for_plotting`, `compare_pae_tf_redshifts`, and `plot_coverage_comparison_grid` functions from `visualization/result_plotting_fns.py`
- Template fitting results are loaded using `load_tf_results` from `data_proc/data_file_utils.py`
- All data indexing and filtering is handled automatically by `prepare_data_for_plotting`
- The coverage plot requires TF PDFs, which take longer to load
- **Burn-in handling**: By default, the first 1000 steps from each MCMC chain are excluded. The summary statistics in the results files already account for burn-in, but the full chains are saved. This script trims burn-in before computing coverage statistics.

## Troubleshooting

**Issue**: "Failed to load data. Check file paths."
- **Solution**: Verify PAE results files exist and have correct permissions

**Issue**: "Coverage plot requires TF PDFs. Use --tf_load_zpdf"
- **Solution**: Add `--tf_load_zpdf` flag when using `--plot_coverage`

**Issue**: Import errors
- **Solution**: Run from `sp-ae-herex/` directory or ensure it's in your Python path
